# Builds on Prior

Evaluates whether the challenge plan correctly accounts for what was taught and built in prior challenges. A challenge that re-introduces already-taught concepts wastes time; one that assumes untaught concepts loses learners. The plan should explicitly distinguish new material from prior knowledge.

## Builds on prior challenges

Evaluate whether the challenge plan is aware of and correctly builds on prior challenge content and infrastructure state.

Score 4: Correctly identifies prerequisites met by prior challenges, existing infrastructure state, and terminology already introduced. New concepts are clearly distinguished from prior knowledge.
Score 5: Precise accounting of prior state. Explicitly marks which concepts get full scaffolding (new) versus confident reference (already taught). Infrastructure changes build on existing state without conflicts.
Score 3: Generally aware of prior context but some assumptions are wrong -- references a service not yet running, or re-explains a concept already covered.
Score 2: References prior challenges but misidentifies what was taught or what infrastructure state exists.
Score 1: No awareness of prior challenges. Re-introduces concepts already taught, or assumes concepts not yet covered.

## Active learning throughout

Evaluate whether the plan ensures learners practice every learning objective rather than having objectives pre-configured by setup scripts.

Score 4: Every challenge section has meaningful planned steps. Learning objectives are practiced by the user, not pre-configured in the environment.
Score 5: Substantive interactive work in every section. All stated objectives require direct user action to accomplish. The plan shows a clear intent to have learners do, not just read.
Score 3: Every section has interactive elements, but some are trivial or token gestures -- "click check to proceed" rather than genuine practice.
Score 2: Most sections have steps, but some are purely instructional with no planned interaction.
Score 1: Multiple sections are passive. Key learning objectives are planned to be pre-configured rather than practiced.

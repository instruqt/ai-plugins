# Learning Objectives

Evaluates whether the track plan's learning objectives are specific, measurable, and actionable. Strong objectives define what a learner can *do* after completing the track -- not what they will "understand" or "learn about." Each objective should map to a concrete skill that can be demonstrated and verified through a check assertion.

## Learning objectives quality

Evaluate whether objectives use action verbs with specific targets that are measurable through challenge steps.

Score 4: Objectives are action-oriented with clear targets (e.g., "configure nginx as a reverse proxy for the backend API"). Each objective maps to observable challenge activities.
Score 5: Every objective is specific, measurable, and includes enough context that an implementer could design a check script to verify it. Objectives collectively describe a coherent skill set, not isolated facts.
Score 3: Objectives start with action verbs but lack specificity (e.g., "configure a service") -- a reader cannot tell exactly what the learner will do.
Score 2: Objectives exist but are unmeasurable or too broad (e.g., "learn about containers"). No clear mapping to challenge activities.
Score 1: No objectives defined, or objectives describe feelings rather than skills (e.g., "understand Git", "appreciate the value of CI/CD").

## Appropriate for audience

Evaluate whether objectives match the stated audience's skill level -- neither too basic for experienced users nor too advanced for beginners.

Score 4: Objectives are calibrated to the stated skill level. A content author could independently judge whether a given explanation is too basic or too advanced based on the objective alone.
Score 5: Objectives are precisely calibrated and explicitly note which skills are new to learn versus prerequisite skills the audience already has. The difficulty gradient is intentional and visible.
Score 3: Objectives are generally appropriate but one or two are noticeably above or below the audience's level without acknowledgment.
Score 2: Several objectives assume skills beyond the stated prerequisites, or are trivially below the audience's level.
Score 1: Pervasive mismatch -- objectives assume expertise not listed in prerequisites, or are so basic they would bore the stated audience.

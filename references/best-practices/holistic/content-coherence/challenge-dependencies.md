# Challenge Dependencies

Each challenge should depend only on concepts taught in prior challenges, forming a clear prerequisite chain.

## Criteria

| Score | Level | Description |
|-------|-------|-------------|
| 1 | Poor | Multiple challenges reference or require concepts that are not taught until later. The prerequisite chain is broken in several places. |
| 2 | Below Standard | One or two challenges assume knowledge from a later challenge. The overall sequence is logical but has noticeable gaps in the dependency chain. |
| 3 | Adequate | The challenge sequence is logical and dependencies flow forward. One minor instance where a concept is referenced slightly before its formal introduction. |
| 4 | Good | All challenges follow a clear prerequisite chain. Every referenced concept has been taught in a prior challenge. |
| 5 | Excellent | The prerequisite chain is airtight. Every concept is taught before it is referenced or required. The sequence has been verified so that each challenge depends only on prior challenges. |

Map the prerequisite chain:
- Challenge N should not require knowledge only taught in Challenge N+2
- If a challenge references a concept, it should be taught in a prior challenge

## What to Watch For

- Challenges that assume knowledge from challenges that come after them

# track.yml

Defines track identity, timing, UI configuration, and ownership. Every track has exactly one `track.yml` at its root.

## Structure

```yaml
slug: my-track-slug
id: <uuid-assigned-by-instruqt>
version: "1.0.0"
type: track
title: "My Track Title"
teaser: "One-line teaser shown in catalog cards"
description: |-
  Multi-line description supporting inline HTML/CSS for brand styling.
  <p style="color: #326CE5;">Styled content is allowed here.</p>
icon: https://example.com/icon.png
level: beginner
tags:
  - owner-slack:team-education
  - hc-product:terraform
  - persona:devops
  - hvd
timelimit: 3600
idle_timeout: 600
show_timer: true
skipping_enabled: true
pausable: false
enhanced_loading: false
maintenance: false
owner: my-org-slug
developers:
  - alice@example.com
  - bob@example.com
instructors:
  - trainer@example.com
lab_config:
  theme:
    name: modern-dark
  default_layout: AssignmentRight
  default_layout_sidebar_size: 33
  override_challenge_layout: false
  sidebar_enabled: true
  extend_ttl: 300
  feedback_recap_enabled: true
  feedback_tab_enabled: false
  loadingMessages: false
  hideStopButton: false
sandbox_preset: ""
checksum: <auto-generated>
```

## Fields

### Top-level identity

| Field | Type | Description |
|-------|------|-------------|
| `slug` | string | URL-safe identifier, must be unique within org |
| `id` | UUID | Assigned by Instruqt, never edit manually |
| `version` | string | Free-form semver string (e.g. `"1.2.0"`) |
| `type` | string | Always `track` |
| `title` | string | Display title in catalog and track player |
| `teaser` | string | One-line summary for catalog cards |
| `description` | string | Multi-line, supports inline HTML/CSS for brand styling |
| `icon` | string | Relative path, HTTPS URL, or inline Unicode/emoji literal |
| `level` | string | `beginner`, `intermediate`, `advanced`, or omitted |
| `checksum` | string | Auto-generated, do not edit |

### Timing

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `timelimit` | int (seconds) | required | Total track time limit |
| `idle_timeout` | int (seconds) | 600 | Seconds of inactivity before cleanup. `0` disables (only for live-presenter tracks). Typical: 300 (5min), 600 (10min), 900 (15min), 3600 (long demos) |
| `show_timer` | bool | false | Show countdown timer to learner. **Generated tracks should set `true`** so learners can pace themselves; hide it only for presenter-paced or open-ended tracks |

### Behavior

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `skipping_enabled` | bool | false | Allow learners to skip challenges |
| `pausable` | bool | false | Enable pause/resume for the track (see operational notes below). **Leave `false` by default** — only enable on explicit user request (e.g. multi-session/multi-day tracks) |
| `pausable_ttl` | int (seconds) | — | Max pause duration. 604800 = 7 days, 1209600 = 14 days. Only set when `pausable: true` |

**Default to `pausable: false`.** Pausing is opt-in: enable it only when the user explicitly asks for a multi-session or multi-day track. **Pausable operational consequences:** When `pausable: true`, the sandbox is suspended and resumed. All state must survive a suspend/resume cycle:
- Avoid tmux, screen, or other terminal multiplexers that hold in-memory state — sessions will be dead after resume
- Avoid in-memory caches (Redis without persistence, local caches) that lose data on suspend
- Use systemd services with `Restart=on-failure` so processes restart after resume
- File-system state, Docker containers with restart policies, and systemd-managed services survive cleanly
- Test the track with an actual pause/resume cycle before publishing
| `enhanced_loading` | bool | — | **Top-level field, NOT under lab_config.** Must be `false` for notes slides to display as overlays |
| `maintenance` | bool | false | `true` hides track from catalog |

### Ownership

| Field | Type | Description |
|-------|------|-------------|
| `owner` | string | Publishing org slug (single value) |
| `developers` | list of strings | Email addresses of developers with edit access |
| `instructors` | list of strings | Email addresses of instructors (separate from developers) |

### Tags

Tags use a structured prefix convention plus standalone flag tags:

**Prefixed tags** (key-value):
- `owner-slack:channel-name` — Slack channel for notifications
- `owner-email:team@example.com` — Contact email
- `hc-product:terraform` — Product association
- `persona:devops` — Target audience

**Flag tags** (standalone):
- `hvd` — HashiCorp Validated Design
- `academy` — Academy track
- `classroom` — Instructor-led
- `time-extension` — Allows time extension
- `skip-testing` — Skip automated testing
- `skipping-disabled` — Disable challenge skipping

### lab_config

Nested under `lab_config:` in the YAML.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `theme.name` | string | `"original"` | `"original"` or `"modern-dark"` |
| `default_layout` | string | `"AssignmentRight"` | `"AssignmentLeft"` or `"AssignmentRight"` |
| `default_layout_sidebar_size` | int | 33 | Sidebar width percentage (20-80). 30 = code-heavy, 40-45 = reading-heavy |
| `override_challenge_layout` | bool | false | Force track-level layout on all challenges |
| `sidebar_enabled` | bool | true | Show sidebar navigation |
| `extend_ttl` | int (seconds) | — | Time extension increment |
| `feedback_recap_enabled` | bool | true | End-of-track summary (low friction) |
| `feedback_tab_enabled` | bool | false | Persistent sidebar feedback tab (more invasive) |
| `loadingMessages` | bool/list | true | `true`, `false`, or list of custom loading messages. **Generated tracks should set `false`** — do not author custom message lists; use notes slides for loading engagement |
| `hideStopButton` | bool | false | Hide the stop button from learners |

### sandbox_preset

Alternative to `config.yml` — provisions a pre-built environment instead of declaring resources.

Preset families include:
- `vendor-a-cloud-native`
- `vendor-b-*`
- `vendor-c-*`
- `vendor-d-*`
- `dev-platform-node-gen2`

When using a sandbox preset, `config.yml` is typically omitted or minimal.

### Layout guidance

- `AssignmentRight` is the default and used by ~80% of tracks
- Typical `default_layout_sidebar_size` values:
  - **30** — code-heavy tracks (more space for terminals/editors)
  - **33** — default balanced layout
  - **40-45** — reading-heavy tracks (more space for instructions)

## Examples

### Minimal track.yml

```yaml
slug: intro-to-terraform
id: abcd1234-5678-90ab-cdef-1234567890ab
version: "1.0.0"
type: track
title: "Introduction to Terraform"
teaser: "Learn the basics of infrastructure as code with Terraform"
description: "Get hands-on with Terraform in this introductory lab."
timelimit: 3600
owner: hashicorp
developers:
  - dev@hashicorp.com
```

### Full-featured track.yml

```yaml
slug: advanced-kubernetes-networking
id: abcd1234-5678-90ab-cdef-1234567890ab

type: track
title: "Advanced Kubernetes Networking"

description: |-
  <img src="https://example.com/k8s-logo.png" width="60" />
  <p>Master advanced Kubernetes networking concepts including:</p>
  <ul>
    <li>Container Network Interface (CNI) plugins</li>
    <li>Service mesh with Istio</li>
    <li>Network policies and segmentation</li>
  </ul>
icon: https://cdn.example.com/k8s-icon.png
level: advanced
tags:
  - owner-slack:team-k8s-edu
  - hc-product:consul
  - persona:platform-engineer
  - classroom

  - k8s-team@hashicorp.com

  - trainer@hashicorp.com

  theme:
    name: modern-dark
  default_layout: AssignmentRight
  default_layout_sidebar_size: 30
  sidebar_enabled: true
  extend_ttl: 600

  loadingMessages: false
  hideStopButton: false
```

# Push Validation Errors

Failure patterns encountered during `instruqt track push`.

## UTF-8 encoding error (null bytes)

**Symptom:** "invalid byte sequence for encoding UTF8: 0x00".

**Cause:** Windows-edited files with trailing null bytes.

**Fix:** Strip null bytes before pushing: `tr -d '\0' < file > file.clean && mv file.clean file`.

---

## Website tab HTTP URL

**Symptom:** Push validation rejects a non-HTTPS website tab URL.

**Cause:** `type: website` tabs require HTTPS URLs.

**Fix:** Change the URL to use `https://`.

---

## Challenge order mismatch

**Symptom:** Push fails or challenges appear in the wrong order.

**Cause:** Directory numbering doesn't match the `track.yml` order.

**Fix:** Ensure sequential `01-`, `02-`, etc. prefixes matching the order in `track.yml`.

---

## Missing required fields

**Symptom:** Push validation error for missing slug, title, etc.

**Cause:** Incomplete `assignment.md` frontmatter.

**Fix:** Ensure all required fields are present (`slug`, `type`, `title`).

---

## Invalid code block syntax

**Symptom:** Run button doesn't appear on code blocks.

**Cause:** Using `` ```bash {"run": "execute"} `` instead of `` ```bash,run ``.

**Fix:** Use comma-separated modifiers after the language identifier (e.g., `` ```bash,run ``).

---

## Checksum mismatch

**Symptom:** Push warning about checksum.

**Cause:** Manually edited checksum field.

**Fix:** Don't edit the checksum -- it's auto-generated by `instruqt track push`.

# Builds on Prior Knowledge

Later content should reference and build on what was taught earlier.

## Criteria

| Score | Level | Description |
|-------|-------|-------------|
| 1 | Poor | Challenges are isolated modules. Earlier concepts are either re-explained in full or ignored entirely. No visible knowledge building between challenges. |
| 2 | Below Standard | Some challenges reference earlier material, but many concepts are re-taught or treated as if the user has not encountered them before. Complexity does not build meaningfully. |
| 3 | Adequate | Later challenges generally build on earlier ones, but there are instances of unnecessary re-explanation or missed opportunities to connect concepts across challenges. |
| 4 | Good | Clear knowledge chain across challenges. Earlier terminology is reused without re-explanation. Complexity increases steadily. One or two connections between challenges could be stronger. |
| 5 | Excellent | Every challenge clearly builds on prior knowledge. Previously taught skills are expected without re-explanation. Complexity increases in a logical, graduated sequence. Concepts compound naturally -- earlier skills are prerequisites for later steps. |

Guidelines for building on prior knowledge:
- Don't re-explain concepts from earlier challenges
- Use terminology that was introduced previously
- Expect users to apply skills learned in earlier assignment steps
- Increase complexity gradually (single file -> multiple files -> branches -> remotes)

Learners retain more when earlier skills resurface in increasingly complex contexts rather than being taught once and left behind:

```
Challenge 1 -- Introduce concept (e.g., staging files)
Challenge 2 -- Apply concept in a focused exercise
Challenge 3 -- Combine concept with a new skill (staging + branching)
Challenge 4 -- Use concept in a realistic, multi-step scenario
```

This pattern is not rigid -- the point is that concepts should compound rather than appear in isolation.

## What to Watch For

- Concepts taught but never built upon later
- Large complexity jumps between challenges (too steep a curve)

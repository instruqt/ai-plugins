# Scaffolding Pattern

The track should follow a scaffolding pattern where concepts progress from modeled to guided to independent.

## Criteria

| Score | Level | Description |
|-------|-------|-------------|
| 1 | Poor | No scaffolding visible. Concepts are either always fully guided or thrown at users without prior modeling. Users never progress to independent application. |
| 2 | Below Standard | Some concepts follow a model-guide-release pattern, but most are either always hand-held or introduced without sufficient initial guidance. The transition is abrupt when it exists. |
| 3 | Adequate | A scaffolding pattern is present for the main concepts, but secondary concepts skip stages (e.g., never modeled before the user is asked to perform them). The release phase is inconsistent. |
| 4 | Good | Most concepts follow a clear model -> guide -> release progression. Users get to practice independently by later challenges. A few concepts could have a smoother transition between stages. |
| 5 | Excellent | Every significant concept follows a deliberate model -> guide -> release arc. Early challenges demonstrate and walk through concepts. Later challenges expect independent application of those same concepts. The transition between stages is smooth and intentional. |

The three stages:
1. **Model**: Show how something works (with explanation)
2. **Guide**: Walk users through doing it (with support)
3. **Release**: Let users do it independently (minimal guidance)

A concept introduced in challenge 2 with full guidance should be expected without re-explanation by challenge 5.

## What to Watch For

- Users never get to practice previously covered skills independently (always hand-held)
- Later challenges still providing the same level of hand-holding as challenge 1 for previously covered skills
